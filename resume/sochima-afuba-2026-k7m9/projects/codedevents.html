Website

What it is

The product is aimed at organizers who need a system for managing the commercial and operational pieces around events rather than only publishing an event page.

What I built

  • Event and administration concepts for setup, access, and operational control.
  • Backend structure with access control, validation, audit logging, and documented API behavior.
  • Deployment paths suitable for Firebase hosting and Cloud Run-backed services.
  • Product framing for ticketing, check-in, vendors, gifting, revenue, and reporting.

Technical surface

The backend work includes Firebase auth, Supabase Postgres, validation layers, Swagger documentation, Docker setup, and Cloud Run deployment paths. The public custom domain is not used here because it is not currently serving the product cleanly.

Product judgment

Events look simple from the outside, but the operator needs control over permissions, money movement, attendance, and records. The product work was to treat those as part of the main experience, not as admin leftovers.

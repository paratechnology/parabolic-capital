MuFit Studio

A real fitness studio business with a public website, member app, admin tools, instructor operations, class booking, credits, purchases, and netball programme workflows.

SectorFitness, wellness, and membership operations
SurfaceWebsite, mobile app, admin, instructor flows, API
StatusLive public website, mobile app
User problem

A fitness studio needs brand presence, class schedules, member booking, payments, credits, attendance, instructors, and programme operations in one flow.

My ownership

I built the website surfaces, member app flows, admin tools, instructor/check-in paths, purchases, credits, netball registration, and server-backed operations.

Hard part

Balancing a consumer-friendly mobile app with staff workflows for capacity, attendance, cancellations, rosters, and member history.

Product views

Studio website first, then the member and staff product.

MuFit is a real fitness business. The software has to support discovery, bookings, credits, classes, netball, staff, and admin work.

MuFit is not just an app concept. It is a business, and the software has to support the website, members, instructors, netball, payments, bookings, and admin work behind it.

Website Mobile app

What it is

The public site presents the studio, classes, schedule, netball, and account journey. The member and staff products handle what happens after interest becomes attendance.

What I built

  • Public website pages for the brand, classes, schedule, account, netball, and supporting content.
  • Member app flows for bookings, cancellations, purchases, credits, profile, class history, and netball registration.
  • Admin and instructor tools for rosters, check-ins, member records, commerce, sessions, and programme management.
  • Server-backed flows for classes, netball sessions, users, payments, push notifications, and account deletion.

Technical surface

The project uses Angular for the web surfaces, Capacitor for mobile packaging, Firebase hosting/services, Firebase Admin on the server, push notifications, and an API hosted at the MuFit domain for class, member, payment, and netball operations.

Product judgment

A studio product has to be clear at the moment of booking and reliable at the moment of attendance. I paid attention to credits, cancellation rules, roster state, instructor visibility, and the difference between a public marketing page and a member operations app.
